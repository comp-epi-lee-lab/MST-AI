Description of Files in the code directory:

01_rgb_mst_ciede2000.py
This script builds the core lookup table used for MST-based skin-tone distance estimation. It converts all possible 8-bit RGB colors into CIELAB space and computes their CIEDE2000 perceptual color distances to the ten Monk Skin Tone (MST) reference swatches. The output is saved as rgb_mst_ciede2000.pckl, which allows later scripts to retrieve MST distances for any skin pixel efficiently without recomputing color-space conversions and CIEDE2000 distances each time.

02_inlier_mst_ciede2000.py
This script processes precomputed inlier skin-region images and summarizes their color similarity to each MST category. For each image, it extracts valid nonzero skin pixels, uses the RGB-to-MST CIEDE2000 lookup table from the previous step, and computes the mean and standard deviation of normalized distances to all ten MST references. It also records the inlier ratio, which reflects how much usable skin area remains after lesion, frame, and outlier removal. The results are saved as inlier_rgb_mst_ciede2000_mu_sig.csv.

03_boltzmann_tau.py
This script tunes the temperature parameter, tau, used in the Boltzmann distribution for converting MST distance summaries into membership probabilities. It reads the per-image mean and standard deviation distances, computes the expected absolute distance for each MST, and sweeps candidate tau values to find the one that produces a target average perplexity of 2.5. In this workflow, the optimal value is reported as approximately tau = 0.0183, which is then used in downstream membership estimation.

04_membership.py
This script converts each image’s MST distance statistics into probabilistic MST membership scores. It reads the mean and standard deviation distances from inlier_rgb_mst_ciede2000_mu_sig.csv, computes expected absolute distances, applies the Boltzmann distribution using the tuned temperature parameter tau = 0.0183, and appends ten probability columns, one for each MST category. The resulting membership probability table is saved as inlier_rgb_membership_probs.csv.

05_cls_inlier.py
This script applies the complete inlier-skin extraction and MST membership pipeline to the classification dataset. For each image, it resizes the image, removes lesion regions using a pretrained segmentation model, detects and removes image frames, isolates the remaining skin pixels, removes color outliers using an Isolation Forest, and creates a combined visualization containing the original image, lesion mask, frame mask, skin image, and final inlier image. It then computes the inlier ratio, MST distance means and standard deviations, and Boltzmann-based MST membership probabilities, saving both per-image CSV files and an updated Excel summary.

05_cls_inlier.out
This is the runtime log for 05_cls_inlier.py. It records that the full classification inlier-processing pipeline finished successfully and took 192218 seconds, indicating that this step was computationally intensive and likely processed a large number of images.

06_stats_plots.py
This script summarizes and visualizes the MST membership outputs generated by the classification pipeline. It filters images based on a minimum inlier-ratio threshold, creates histograms of inlier ratios and MST membership probabilities, computes rank-based and elbow-based MST assignments, saves several interactive Plotly HTML plots, and exports an updated classification_ranks.xlsx file with binary MST membership indicators. It also generates benign-versus-malignant distribution plots across MST categories to support subgroup-level dataset analysis.

06_stats_plots.out
This is the runtime output from 06_stats_plots.py. It reports that 879 samples were below the inlier-ratio threshold and 73,327 were retained above the threshold. It also prints the total, benign, and malignant sample counts for each MST category, showing strong imbalance across MST groups; for example, MST 2 has 36,166 total samples, while MST 10 has only 144. The script finished in 220 seconds.

07_make_tvt.py
This script creates train, validation, and test splits from the MST-ranked classification dataset. It uses the binary MST membership columns and class labels to assign samples into train, validation, and test subsets while prioritizing MST categories with fewer samples first. The split is stratified by benign/malignant labels when possible, and the final table includes an added tvt column indicating the assigned subset. The outputs are saved as tvt.xlsx and tvt.pckl for later model training and evaluation.

check_images_availability.py
This utility script checks whether all image paths listed in the classification Excel file exist on disk. It reads classification.xlsx, iterates through the image column, and prints any missing file paths. This is a quality-control step to confirm that dataset metadata and image storage paths are correctly aligned before running computationally expensive preprocessing or modeling steps.

look_at_some_images.py
This utility script copies selected processed images from one MST subgroup into a separate folder for visual inspection. In its current configuration, it loads the train/validation/test dataset table, selects images assigned to mst_9, and copies their combined inlier-visualization images into an output directory. This helps manually review whether the MST assignment and inlier extraction outputs look reasonable for a specific skin-tone subgroup.

make_classification_256x256.py
This script extracts the original 256×256 image panel from the combined inlier-visualization images generated by the classification pipeline. Each combined image contains multiple panels side by side, and this script crops the first 256-pixel-wide panel, corresponding to the resized original dermoscopic image, then saves it into a dedicated classification image directory. This produces clean 256×256 classification inputs for downstream model training.
